# 🏞️ 景区多智能体系统：基于 Dify 知识库与 LangGraph+CrewAI 的落地案例

## 1. 项目背景与业务概述
本项目旨在为现代景区打造一个全链路、全天候的智能化服务中枢。通过引入大模型与多智能体（Multi-Agent）技术，系统不仅能为游客提供购票、导游、研学、文创购买等一站式服务，还能为景区运营方提供文案生成、数据分析等后台支持，实现降本增效与游客体验的全面升级。

## 2. 核心框架结构
本项目采用了 **LangGraph + CrewAI** 的双核驱动架构，搭配 **FastAPI** 作为高性能网关，构建了高扩展性的底层系统：

*   **接入与网关层 (FastAPI)**：统一处理来自微信小程序、公众号、抖音及 Web 端的请求，负责鉴权、限流与基础路由。
*   **中枢控制大脑 (LangGraph)**：系统的“指挥官”。负责全局的状态管理（Stateful）、长短期记忆（Memory）、意图路由（Workflow Routing）以及在关键节点引入人工审核（Human-in-the-loop），确保流程可控。
*   **多智能体协作团队 (CrewAI)**：执行层的“专家团”。由6个高度垂直的智能体组成（客服、文案、研学、票务、文创导购、管理后台）。LangGraph 将复杂任务拆解后，调度给特定的 CrewAI 智能体进行协作。

## 3. Dify 知识库 (RAG) 的深度应用（核心亮点）
大模型本身缺乏景区的实时信息和私有知识。本项目通过深度集成 **Dify RAG（检索增强生成）**，将其作为整个系统的“企业级知识引擎”，彻底解决 AI 幻觉问题。

### 3.1 知识库内容资产划分
在 Dify 中，我们将景区数据结构化为三个核心知识库：
1.  **景区百科与问答库**：包含景区历史、景点介绍、开放时间、交通指南、常见 QA 等。
2.  **研学教育知识库**：包含历史文化深度讲解、动植物科普资料、互动课程题库。
3.  **营销与品牌资产库**：包含历史优秀文案、品牌语料、活动策划模板。

### 3.2 智能体与 Dify 的联动场景案例
*   **场景一：客服智能体 (D1) 的精准问答**
    *   **游客提问**：“今天下午有实景演出吗？在哪里看？”
    *   **执行流程**：网关接收请求 -> LangGraph 路由判定为“咨询意图” -> 唤醒客服智能体 -> 客服智能体调用 **Dify API** 检索最新演出时间表 -> 结合上下文生成拟人化回复。
    *   **价值**：保证了票务、演出等时效性信息的绝对准确。

*   **场景二：研学智能体 (D3) 的互动式教学**
    *   **游客提问**：“这个古建筑有什么历史典故？”
    *   **执行流程**：LangGraph 唤醒研学智能体 -> 调用 **Dify 研学知识库**提取该景点的文化背景 -> 智能体将其转化为适合学生年龄段的生动讲解，并附带一个互动小问题。
    *   **价值**：实现了千人千面的个性化数字导游服务。

*   **场景三：文案智能体 (D2) 的自动化营销**
    *   **运营指令**：“生成一篇五一假期的朋友圈推广文案。”
    *   **执行流程**：文案智能体从 **Dify** 中检索“节假日营销模板”和“近期主推景点” -> 生成多套风格（文艺、活泼、幽默）的文案。
    *   **价值**：结合 Coze / n8n，甚至可以实现文案生成后自动分发至各大社交平台。

## 4. 工具链与自动化执行 (Action)
为了让智能体不仅能“说”，还能“做”，系统接入了强大的工具层：
*   **OpenClaw / QClaw 工具集**：赋予智能体操作系统 API 的能力。例如，票务智能体 (D4) 可以直接调用景区票务系统和微信支付接口完成扣款与出票；文创导购智能体 (D5) 可以操作电商购物车。
*   **Coze / n8n 自动化流**：处理定时任务和复杂的自动化流水线，如每天定时拉取运营数据生成日报。

## 5. 架构优势与业务价值总结
1.  **高可靠性（知识不越界）**：依托 Dify 的 RAG 能力，客服和研学回答严格锚定景区真实数据。
2.  **复杂任务的解耦**：通过 LangGraph 的中枢路由，不同任务交由 CrewAI 中专门的 Agent 负责，系统逻辑清晰，互不干扰。
3.  **极强的可扩展性**：未来若需新增“餐饮推荐智能体”，只需在 CrewAI 中注册新角色，并在 Dify 中导入餐饮知识库即可，底层架构无需任何改动。
